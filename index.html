// features
⚡

select() Multiplexing

Single-threaded server monitors all client sockets and the server socket in one select() call — no thread-per-client, no fork overhead.

👥

Up to 20 Clients

Supports 20 concurrent connections via a fixed-size file descriptor table. Each client gets a username-keyed slot.

🔑

Unique Usernames

On connect, server prompts for a username and re-prompts until a unique one is provided — no two active users can share a name.

📡

Broadcast Messaging

Every message is forwarded to all other connected clients with the sender's username prepended. Sender does not receive their own message.

🔌

Clean Disconnect

Typing exit removes the client from the FD set, compacts the descriptor array, and logs the disconnect on the server terminal.

🖥️

Responsive Client

Client also uses select() to monitor stdin and the socket simultaneously — prints incoming messages without interrupting user input.

// architecture
  ┌─────────────────────────────────────────────────────────────┐
  │                        SERVER                               │
  │   select() loop — monitored FDs:                            │
  │     serverSocket   ←  new TCP connections (accept)          │
  │     fileDesc[0..N] ←  messages from connected clients       │
  │                                                             │
  │   On new connection:                                        │
  │     accept() → send "Enter username:" → recv username       │
  │     validate uniqueness → store in userNames[] + fileDesc[] │
  │                                                             │
  │   On message received:                                      │
  │     recv() → prepend "username: " → broadcast to all others │
  │     if message == "exit" → FD_CLR + compact + close         │
  └──────────────────────┬──────────────────────────────────────┘
                         │  TCP · port 1026
          ┌──────────────┴──────────────┐
          │                             │
   ┌──────▼──────┐               ┌──────▼──────┐
   │  Client A   │     . . .     │  Client N   │   (up to 20)
   │  select():  │               │  select():  │
   │  stdin  ──► send()          │  stdin  ──► send()
   │  socket ──► print msg       │  socket ──► print msg
   └─────────────┘               └─────────────┘
// interactive demo
server — bash
client: alice — bash
client: bob — bash
// build & run
# Clone the repository
git clone https://github.com/Halkhoori2000/TCP-Chat-Server.git
cd TCP-Chat-Server/src

# Compile (requires GCC on Linux)
gcc -o server server.c
gcc -o client client.c

# Terminal 1 — start the server
./server
# Listening...

# Terminal 2 — connect as alice
./client
# Enter username: alice

# Terminal 3 — connect as bob
./client
# Enter username: bob
Note: The source uses hardcoded IP 10.0.2.15 (VirtualBox NAT default). To run locally, replace it with 127.0.0.1 in both client.c and server.c.
